The Goal

spark-ai-stack is a fully self-hosted AI content pipeline running on an NVIDIA DGX Spark. Send a tech topic through Telegram, let a local Gemma 4 model write the script, render animations with Manim, generate narration with Kokoro TTS, merge the assets with FFmpeg, and send the finished video back to Telegram for review.

The design principle is simple: no hosted model APIs, no per-video inference cost, and no handoff between separate tools once the workflow starts.

User Flow

01Prompt in Telegram

Send a topic like "Python decorators" to the Telegram bot.

02n8n validates the request

The workflow sends a working message, asks Gemma 4 whether the topic is technical, and rejects off-topic requests.

03Gemma 4 writes the script

The local model produces narration plus scene instructions using a constrained animation schema.

04Render server builds assets

A Flask service turns scene data into Manim animations and Kokoro voiceover.

05FFmpeg merges the video

Rendered animation and audio are composed into a final MP4.

06Telegram returns review copy

The finished short lands back in Telegram, ready to inspect before upload.

Render Reliability

The key reliability decision is that Gemma does not write raw Manim code. The model chooses from 12 pre-built animation templates and provides the data each template needs. That keeps generation creative at the script layer while making rendering deterministic.

Operations

Daily operation is intentionally simple: start Docker services, run the render server, open the Cloudflare tunnel, and warm the model to avoid cold-start delay.

docker compose up -d python3 render-server/server.py cloudflared tunnel --url http://localhost:5678 docker exec -it ollama ollama run gemma4:26b "hi"
