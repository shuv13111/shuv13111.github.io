The Problem

Supply-chain analysts fielded roughly 100 ad-hoc questions a week — "which SKUs are exposed if this supplier slips?", "what's the alternate routing for this lane?" Each answer meant manually joining data across systems: an average of 6 hours per question, often longer.

The knowledge existed. It was just scattered across millions of entities and relationships no single query could reach.

The Approach

Model the supply chain as a knowledge graph, then let an agent traverse it. I designed a Neo4j graph of 220M nodes and 80M edges — suppliers, SKUs, lanes, facilities, purchase orders — and built a React and FastAPI application around a LangGraph agent that uses Gemini and Claude APIs, calls Neo4j tools through MCP, validates intermediate results, and composes a sourced answer.

  • Agentic query planning: the agent decomposes questions into Cypher traversals and self-checks results
  • Tool access through MCP: Neo4j tools expose controlled graph retrieval to the LangGraph workflow
  • Retrieval grounded in the graph — every claim in an answer traces to specific nodes
  • Latency budget enforced per step; 38s average end-to-end
